1//! # claude-code-sync
2//!
3//! A command-line tool for synchronizing Claude Code conversation history with Git repositories.
4//!
5//! ## Overview
6//!
7//! `claude-code-sync` enables you to back up, version control, and share your Claude Code conversation
8//! history across multiple machines. It works by syncing conversation files (stored as JSONL) from
9//! your local Claude Code projects directory (`~/.claude/projects/`) to a Git repository, which
10//! can be pushed to a remote server for backup or collaboration.
11//!
12//! ## Key Features
13//!
14//! - **Bidirectional sync**: Push local conversations to a Git repository and pull remote changes
15//! - **Conflict detection**: Automatically detects when the same conversation has diverged
16//! - **Interactive resolution**: Provides a TUI for resolving conflicts interactively
17//! - **Filtering**: Exclude attachments, old conversations, or specific projects
18//! - **Operation history**: Track all sync operations with automatic snapshots for undo
19//! - **Cross-platform**: Supports Linux, macOS, and Windows with platform-specific config directories
20//!
21//! ## Architecture
22//!
23//! The library is organized into modules that handle different aspects of the sync process:
24//!
25//! - Configuration and state management ([`config`], [`filter`])
26//! - Git operations ([`git`])
27//! - Conversation parsing and analysis ([`parser`])
28//! - Conflict detection and resolution ([`conflict`])
29//! - Operation tracking and undo ([`history`], [`undo`])
30//! - User interface and reporting ([`onboarding`], [`report`], [`logger`])
31//! - Core synchronization logic ([`sync`])
32

41/// Platform-agnostic configuration directory management for claude-code-sync.
42///
43/// Provides utilities for locating and managing configuration files and directories
44/// following platform conventions (XDG on Linux, Application Support on macOS,
45/// AppData on Windows).
46pub mod config;
47
48/// Conflict detection and resolution for conversation synchronization.
49///
50/// Detects when the same conversation has diverged between local and remote copies
51/// by comparing content hashes. Provides strategies for resolving conflicts including
52/// keeping both versions (with automatic renaming), keeping local, or keeping remote.
53pub mod conflict;
54
55/// Interactive terminal-based conflict resolution interface.
56///
57/// Provides a user-friendly TUI for resolving sync conflicts interactively. Users can
58/// view detailed comparisons of conflicting conversations and choose resolution strategies
59/// (keep local, keep remote, or keep both) on a per-conflict basis.
60pub mod interactive_conflict;
61
62/// File filtering configuration for selective synchronization.
63///
64/// Controls which conversation files are included in sync operations based on
65/// criteria such as file age, path patterns, file size, and file type (e.g.,
66/// excluding attachments to sync only JSONL conversation files).
67pub mod filter;
68
69/// Git repository operations for conversation history management.
70///
71/// Provides a high-level interface to libgit2 for common Git operations including
72/// initializing repositories, cloning, committing, pushing, pulling, and fetching.
73/// Handles authentication via Git credential helpers and SSH agents.
74pub mod git;
75
76/// Operation history tracking and persistence.
77///
78/// Records all sync operations (push and pull) with metadata about affected
79/// conversations. Maintains a rolling history of recent operations with automatic
80/// rotation. Each operation record includes a snapshot path for undo functionality.
81pub mod history;
82
83/// Logging configuration and utilities.
84///
85/// Sets up dual logging to both console (configurable via `RUST_LOG` environment
86/// variable) and a persistent log file in the config directory. Includes automatic
87/// log rotation when files exceed size limits.
88pub mod logger;
89
90/// Smart merge functionality for combining divergent conversation branches.
91///
92/// Provides intelligent merging of conversation sessions by analyzing message UUIDs,
93/// parent relationships, and timestamps. Can handle non-overlapping messages, edited
94/// messages (resolved by timestamp), conversation branches (all branches preserved),
95/// and entries without UUIDs (merged by timestamp).
96pub mod merge;
97
98/// Interactive onboarding flow for first-time setup.
99///
100/// Guides users through initial configuration including repository setup (clone vs local),
101/// remote URL configuration, and filter preferences. Uses terminal UI prompts to collect
102/// user preferences and validates inputs before saving configuration.
103pub mod onboarding;
104
105/// JSONL conversation file parsing and serialization.
106///
107/// Parses Claude Code conversation files (JSONL format) into structured data.
108/// Each conversation session contains multiple entries (user messages, assistant responses,
109/// file snapshots, etc.) with metadata like timestamps, UUIDs, and session IDs.
110pub mod parser;
111
112/// Conflict report generation and formatting.
113///
114/// Generates detailed reports of sync conflicts in multiple formats (JSON, Markdown, console).
115/// Reports include information about diverged conversations, message counts, timestamps,
116/// and resolution strategies applied during the last sync operation.
117pub mod report;
118
119/// Core synchronization logic for pushing and pulling conversation history.
120///
121/// Implements the main sync operations:
122/// - **Push**: Copies local conversations to the sync repository and commits changes
123/// - **Pull**: Merges remote conversations into local storage with conflict detection
124/// - **Sync**: Bidirectional operation that pulls then pushes for full synchronization
125///
126/// Includes state management, session discovery, conflict handling, and operation tracking.
127pub mod sync;
128
129/// Snapshot-based undo functionality for sync operations.
130///
131/// Creates point-in-time snapshots of conversation files before sync operations.
132/// Snapshots enable undoing pull operations (by restoring files) and push operations
133/// (by resetting Git commits). Includes validation and security checks for safe restoration.
134pub mod undo;
